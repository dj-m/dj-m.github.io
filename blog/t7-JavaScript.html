Javascript vs Ruby

August 2, 2015

So, how do Javascript and Ruby differ? Well, let me count the ways...

Javascript documentation vs Ruby documentation

  • If the variable begins with '$' it's a global variable
  • If the variable begins with '@' it's an instance variable
  • If the variable begins with [a-z] or '_' it's a local variable
  • If the variable begins with [A-Z] it's a constant
  • If the variable begins with '@@' it's a class variable

Global Variables

These variables are accessible from anywhere in the Ruby program, regardless of where they are declared. Global variable names are prefixed with a dollar sign '$'.

Instance Variables

Instance variables are similar to Class variables except that their values are local to specific instances of an object. nstance variables are declared in Ruby by prefixing the variable name with a single '@' sign

Class Variables

A class variable is a variable that is shared amongst all instances of a class. This means that only one variable value exists for all objects instantiated from this class.Class variables are declared by prefixing the variable name with two @ characters '@@'.

Constant Scope

Ruby constants are values which, once assigned a value, should not be changed. Constants declared within a class or module are available anywhere within the context of that class or module. Constants declared outside of a class or module are assigned global scope.

::: Quick Tip! :::

Should you ever want to find out the scope of any particular variable just run this bit of code: "defined?" followd by the variabnle. It's super useful!
