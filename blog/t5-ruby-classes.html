Ruby Classes! Let the fun begin, right?

July 21, 2015

Here we go... do you want to create an array? Does each elements need to be populated with some specific data? Do you like typing as little as possible? Well, may I have the pleasure of introducing you to the 'map' method of Ruby?

That was mostly rhetorical, but hereâ€™s how this'll shake out:

(1..4).map {|x| x * x}

This bit of code will build an array populated with elements that go from 1 to 4 where each one of those elements will be multiplied by itself. You'll get the following output:

#=> [1, 4, 9, 16]

So the 'map' method here returns a new array filled with whatever gets returned by the block each time it runs.

The funny thing about the #map method is that it's the EXACT same method as #collect... but just called differently. Which you use is a matter of how you envision the data moving. So, some people view the resulting array as collecting the results while others view it as a transformation or remapping of data.
